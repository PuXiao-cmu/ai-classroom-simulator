# Three-minute demo script

## 0:00–0:25 — The promise

“Teachers usually practice what they’ll say, but not how five different minds might receive it. Classroom Lens lets a K–12 STEM teacher rehearse the learning—not just the presentation.”

Click **Enter the demo classroom**. Briefly scan the five student cards and point out that visible profiles never reveal hidden mastery or misconceptions.

## 0:25–1:25 — The magic loop

Start the rehearsal. Use the three demo teaching-path controls or say these lines:

1. “When a truck and a bicycle collide, which one experiences the greater force?”
2. After Maya’s answer: “The forces are equal and opposite, even though the bicycle accelerates more because it has less mass. Each force acts on a different object.”
3. “Maya, try the truck and bicycle question again. Which force is greater, and why?”

Call out the controlled hand-raise anticipation, one-speaker state, and Maya’s changed later answer. Do not show internal state during the live class.

## 1:25–2:15 — Classroom X-Ray

End the rehearsal. Show:

- the number of bounded learning shifts;
- feedback moments linked to timestamps;
- Maya’s learning journey;
- the evidence drawer containing surrounding transcript events rather than invented quotes.

## 2:15–2:50 — Why it is technically different

“This is not five chatbots. One GPT-5.6 orchestrator proposes small, evidence-linked state changes. Deterministic code validates and commits them. The student’s next behavior is generated from the updated state, so learning is explicit and inspectable.”

Mention GPT-5.6 Terra for preparation/report quality, Luna for the latency-sensitive live path, Realtime transcription over WebRTC, and distinct OpenAI TTS voices.

## 2:50–3:00 — Close

“Classroom Lens helps teachers find the explanation that unlocks the room—before real students are waiting.”

On-screen final note: camera and audio are not recorded; the simulation is a rehearsal hypothesis, not a prediction about real students.
